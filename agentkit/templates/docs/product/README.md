# Product

- **Purpose:** Define *what* is being built and *why* — problem, users, goals,
  scope, success criteria. The most upstream documents.
- **Document types:** PRD, product brief, roadmap.
- **Naming:** `prd.md` for the primary PRD; `<topic>-brief.md` for briefs.
- **Inputs / outputs:** starting point; feeds `requirements/`.

Keep PRDs lightweight ("PRD-lite") unless scope is large. Start from `prd.template.md`.
