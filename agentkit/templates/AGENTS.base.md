# Repository Guidelines

## Canonical Agent Context

`AGENTS.md` and `.agents/**` are the single source of truth for repository agent
instructions, project-local skills, MCP server metadata, permission rules, and
provider gap notes. Edit those; do not edit generated provider adapters.

Adapters are produced by agentkit and carry a "Generated by agentkit" marker. After
changing agent context, run `agentkit sync` and `agentkit validate`.

Provider precedence: user prompt > local/private provider config > generated adapter
> canonical repo source. Generated adapters must not weaken repo safety rules. Keep
credentials local in `.env.mcp` for MCP; committed config may contain environment
variable names only, never secret values.

## Skills

Project-local skills live under `.agents/skills` and are exposed to providers via
generated symlinks (`.kiro/skills`, `.claude/skills`) and discovered natively by
opencode. They are grouped by name prefix (see `.agents/README.md`):

- `core-*` — agent operations: `core-init` (bootstrap/adapt), `core-consultant`
  (reason/recommend before a decision), `core-orchestrator` (route/plan/execute).
- `virtual-assistant-*` — job-function methodology (BA, architect, developer, …).
- `stack-*` — technology conventions for this project's stack.

## Intake

For a broad or unclear request, start with `core-consultant` to frame the problem
and decide, then `core-orchestrator` to route to the right skill and plan. For a
small, clear task, pick the relevant skill and proceed. If a relevant MCP/tool is
unavailable, fall back to local inspection, local commands, focused tests, or state
the blocker.

## Context7 Documentation

Use Context7 MCP to fetch current documentation whenever a user asks about a
library, framework, SDK, API, CLI tool, or cloud service — even well-known ones,
because local knowledge may be stale. Resolve the library ID first (unless the user
gives an exact `/org/project` ID), then query docs and answer from them.

## Security & Configuration

Do not commit secrets, local credentials, or machine-specific paths; use environment
variables or external configuration and keep safe defaults out of version control.
Copy `.env.mcp.example` to `.env.mcp` for local agent/MCP credentials only.

## Confirmation Priority

Ask for explicit confirmation before Git history or remote operations, destructive
filesystem actions, credential handling, staging/prod changes, deploys, or releases.
Proceed without extra confirmation for repository inspection, local tests, local
builds, local artifact generators, and small clear edits within the requested scope.
